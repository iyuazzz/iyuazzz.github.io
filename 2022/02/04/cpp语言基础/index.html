c++编译过程

  • 预处理阶段
    • 消除注释
    • 替换宏定义
    • 处理条件编译命令
    • 生成预编译文件 *.cpp -> *.i
  • 编译阶段
    • 语法分析与词法分析
    • 翻译成汇编代码 *.i -> *.s
  • 汇编阶段
    • 将汇编语言翻译成目标机器指令
    • 生成目标文件 *.s -> *.o
    • 目标文件由两段组成
      • 代码段 : 包含主要程序指令， 可读可执行，不可写
      • 数据段 : 全局变量或静态书读， 可读可写可执行
  • 链接阶段
    • 将多个目标文件以及所需要的库链接成最终可执行目标文件
    • *.o -> 可执行文件

sizeof(指针)与sizeof(数组)

  • 指针
    • 返回指针所指向的对象的字节大小
  • 数组
    • 返回数组所有元素的字节大小总和
  • 编译器会自动辨别参数是指针还是数组首地址
  • 数组作为参数传进子函数内时，会被sizeof当作指针处理

c++内存区

  • 栈区
    • 编译器需要的时候就分配，不需要时自动释放，如局部变量，函数参数等
  • 堆区
    • 用于动态内存分配， 一般手动分配或释放， 若未释放，程序结束时可能由操作系统回收， 例如malloc, free
  • 静态区
    • 一旦被分配，直到程序结束才会被释放， 如全局变量
  • 代码区
    • 存放你代码编译出的可执行文件，只读属性， 但可以执行
  • 常量存储区
    • 存放常量，不可修改
  • 自由储存区
    • 由new分配出的内存，是一种抽象概念

new与malloc的区别

  • 性质不同
    • new/delete是c++的关键字，(底层调用operator new操作符) 需要编译器支持
    • malloc/free是库函数， 需要头文件malloc.h支持
  • 内存区域不同
    • malloc函数从堆上动态分配内存
    • new操作符从自由存储区为对象动态分配内存
      • 堆是操作系统维护的一块内存，是物理概念
      • 自由存储区是c++中通过new与delete动态分配对象的存储区，是逻辑概念
      • 只要是通过new操作符申请出的内存，就叫自由存储区
      • new操作符底层默认使用malloc申请内存，所以自由存储区就位于堆上，但是new操作符是可以重载的，改用其他内存来分配，(比如全局变量做内存池), 此时自由存储区就不在位于堆上了
  • 能否重载
    • c++允许重载 new/delete 操作符
    • mallock/free函数不可重载
  • 细节
    • 语法
      • new
      • malloc
      • delete与free
        • delete相比于free， 还会在释放内存之前先析构指针所指向的对象
    • 构造函数
      • new操作符申请完内存后，会调用类型的构造函数初始化成员变量
      • malloc仅仅是申请出内存，并不关心内存的数据类型
    • 参数
      • 使用new操作符申请内存，无需指定内存块大小， 编译器会根据数据类型自动推算
      • 使用malloc申请内存需要显示指出所需内存块的大小， 可使用 sizeof计算每种数据类型的大小
    • 返回值
      • new分配成功时，返回类型严格与对象匹配的指针
      • malloc分配成功时，返回void *， 需要强制类型转换成我们需要的类型
      • new分配失败时, 会抛出bac_alloc异常
      • malloc分配失败时， 会返回NULL
