Git笔记

Posted on 2022-02-08

方便学习Git

概念

版本控制系统

简而言之， 就是记录文件每次改动， 自由切换不同版本

分为集中式与分布式两种版本控制系统

  • 集中式

    • 版本库集中存放在中央服务器
    • 工作时, 先从中央服务器获取最新版本到本地电脑， 结束后把本地的新版本推送到中央服务器
    • 必须联网
    • 常见的有SVN与CVS

  • 分布式

    • 没有中央服务器， 每个人电脑就是一个版本库
    • 无需联网
    • 常见的有Git

Git结构

主要分为工作区, 版本库

  • 工作区
    • 电脑里你能看见的目录
  • 版本库
    • 工作区有个隐藏目录.git就是版本库
    • 版本库内有暂存区stage, 分支与HEAD指针
    • 使用git add指令可以将文件提交到暂存区
    • 使用git commit指令可以将暂存区内容推到分支上
    • HEAD指针指向分支的某个节点, 为当前版本

命令

  • git init

    • 把当前目录变成Git的一个仓库

  • git add + [filename]

    • 把本地指定文件提交到缓存区， 如git add a.cpp
    • 该文件必须在此仓库的目录下（子目录也可以)

  • git commit -m "[message]"

    • 将缓存区的所有文件提交到本地仓库
    • -m message为本次提交的说明
    • git commit -m "test"

  • git log

    • 查看历史提交记录

  • git reset -- hard + [id]

    • 回退到提交号为id的版本
    • 若不记得id 可用 git log 查找
    • 如git reset --hard 1094a

  • git status

    • 查看当前分支下仓库的状态

  • git diff

    • 比较文件之间不同

  • git rm + [filename]

    • 删除仓库内指定文件

  • git remote add origin git@github.com:username/reponame/git

    • 连接远程仓库, username替换成该仓库所有者名称， reponame为该仓库名称
    • origin为自定义的远程库名称，默认写origin

  • git push origin master

    • 把本地仓库推送到远程仓库的master分支上
    • 首次推送需要在push后加上-u参数

  • git remote -v

    • 查看远程库信息

  • git remote rm origin

    • 断开与远程库origin的连接

  • git clone [name]

    • 将远程库克隆到本地
    • name为远程库地址

  • git branch

    • 查看当前仓库分支

  • git merge [name]

    • 将指定分支合并到当前分支（并不会删除分支）
    • name为指定的分支名称

  • git switch [name]

    • 切换到指定分支
    • 在 switch后加上-c参数可以创建并切换

  • git btanch -d [name]

    • 删除指定分支
